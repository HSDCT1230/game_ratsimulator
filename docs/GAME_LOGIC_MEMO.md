# 鼠鼠模拟器规则与数值备忘录

更新时间：2026-07-08

本文档记录 `index.html` 中当前实际生效的游戏规则和数值。后续改玩法时，先改代码，再同步这份备忘录，避免玩家看到的规则和真实结算不一致。

## 1. 平衡目标

- 笼子是安全恢复区，适合补饱腹、降出恭、睡觉、整理窝、梳毛和跑轮。
- 客厅是低风险缓冲区，是所有外出地点的第一站，风险低于厨房、阳台和门外。
- 厨房偏食物和储备收益，但主人发现概率最高。
- 阳台偏闲游、晒太阳和轻度磨牙，风险中等，主要危险是中暑、着凉、晾衣杆滑落、植物、边缘滑落和窗帘线缠绕。
- 门外是高收益高风险探索区，有高额食物/磨牙/出恭收益，也有猫、拖鞋、水淹、冻伤等风险。
- 危险事件优先走“轻伤/逃过一劫”，同一天重复触发同类危险才更可能致命，减少“一次随机直接结束”的挫败感。
- 本轮数值节奏：吃饭次数略少但单次更有效，磨牙次数更紧张，闲游次数更多；玩家会更自然地在客厅、厨房、阳台、门外之间选择路线，而不是只在笼子里重复点安全按钮。

## 2. 基础状态

- 游戏从 `6:00` 开始，初始位置为 `笼子`。
- 五项生命值统一限制在 `0-100`：饱腹、磨牙、闲游、抚触、出恭。
- 初始生命值：饱腹 `50`、磨牙 `50`、闲游 `50`、抚触 `50`、出恭 `10`。
- 初始资源：食物储备 `150`、主人信任 `80`。
- 初始每日次数：吃饭 `11`、磨牙 `6`、闲游 `13`、抚触 `11`、电话 `2`。
- 每局重开会随机切换鼠鼠档案、主人、邻居鼠和日记索引；累计战绩会保留。
- 胜利条件：存活天数 `> 3` 时通关并停止计时。

## 2.1 参数、字段与计算公式说明

本节解释代码里常见字段和公式的含义。除特殊说明外，百分比类数值都按 `0-100` 保存，低值类越低越危险，出恭越高越危险。

### 2.1.4 计算公式

| 函数/公式 | 作用 | 计算方式 |
| --- | --- | --- |
| `clampPercent(x)` | 限制百分比 | 把属性限制在 `0-100`，避免进度条溢出 |
| `scaledAmount(value, multiplier, minValue)` | 按倍率计算整数收益/损失 | `raw = value * multiplier`；整数部分直接保留，小数部分按概率进位；最后不低于 `minValue` |
| `scaledChance(baseChance, multiplier, maxChance)` | 按倍率计算概率 | `baseChance * multiplier` 后限制在 `0-maxChance` |
| `passesChance(baseChance, multiplier, maxChance)` | 判断事件是否发生 | 随机数 `< scaledChance(...)` 则触发 |
| `randomGain(min, max, multiplier)` | 随机收益/损失 | 先在 `min-max` 间取整数，再走 `scaledAmount(..., multiplier, 1)` |
| `getDailyFoodStock(baseStock)` | 新一天补粮 | `max(75, baseStock - foodPenalty)` |
| `getDailyLoveLimit(baseLimit)` | 新一天抚触次数 | `max(5, baseLimit - lovePenalty)` |
| `changeOwnerTrust(delta, reason)` | 调整主人信任 | `ownerTrust + delta` 后限制在 `0-100` |
| 主人发现概率 | 普通被发现 | `3.5% * ownerRisk * 信任discoveryRisk * 地点discoveryRisk`，最高 `28%` |
| 厨房发现概率 | 厨房高风险被发现 | `4.5% * 1.28 * ownerRisk * 信任discoveryRisk * 厨房discoveryRisk`，最高 `42%` |
| 禁足时长 | 抓包后的外出限制 | 当天第 `1` 次抓包加 `2` 小时；重复抓包加 `3` 小时；总剩余最高 `6` 小时 |

`scaledAmount` 的小数进位用于避免“倍率后的小数被永久吃掉”。例如基础收益 `13`、倍率 `1.18` 时，`raw=15.34`，实际通常给 `15`，有 `34%` 概率给 `16`，长期平均接近 `15.34`。

### 2.1.5 风险事件配置字段

多数意外风险会用同一套字段配置，格式类似 `applyActionDanger({...})` 或 `resolveDangerEvent(type, deathType, options)`。

| 字段 | 含义 |
| --- | --- |
| `key` / `type` | 今日危险记忆键；同一天同一个键第二次触发时，会进入“重复危险”判断 |
| `deathType` | 致命时使用的死亡类型；也会影响默认死亡文案和默认音效 |
| `chance` | 本次危险基础概率 |
| `multiplier` | 危险概率倍率，通常来自鼠鼠性格 |
| `maxChance` | 概率上限，防止倍率叠加后过高 |
| `fatalChance` | 今日同类危险已经轻伤过后，再次触发时的致命概率 |
| `foodLoss/chewLoss/playLoss/loveLoss` | 轻伤时扣除的属性 |
| `poopGain` | 轻伤时增加的出恭值，常用于受惊、肚子不舒服等事件 |
| `icon/title/popupDesc` | 第一次轻伤弹窗展示内容；`popupDesc` 必须是非数值文案，不写则使用通用提示 |
| `repeatTitle/repeatPopupDesc` | 同日重复轻伤弹窗展示内容；若进入致命判定成功则直接死亡 |
| `sfx` | 触发轻伤时播放的音效；不写时会优先用 `deathType` 对应的音效 |
| `log` | 写入底部日志的文本；具体扣值会由结算函数自动追加到日志 |

惩罚提醒界面不展示具体数值。轻伤、主人抓包、中毒、着凉、迷路、电击等弹窗只提示发生了什么；具体的饱腹、磨牙、闲游、抚触、出恭、信任变化必须只出现在底部日志栏。

常见 `key` 示例：`wireElectrocution` 表示严重触电，`wireShock` 表示轻微触电（不致命），`sewer` 表示下水道危险，`clothesRodFall` 表示爬晾衣杆滑落，`curtainTangle` 表示窗帘线缠住，`plantPoison` 表示植物/叶子中毒，`wheelStuck` 表示跑轮卡脚/尾巴。**不同伤害类型必须使用不同key**，即使发生在同一位置；`key` 越细，同一天重复惩罚越精确；不同 `key` 不会互相触发重复致命判定。

风险结算顺序一般是：

1. 行动先消耗对应次数，例如闲游次数 `-1`。
2. 判断危险是否触发：`passesChance(chance, multiplier, maxChance)`。
3. 若未触发危险，发放正常收益。
4. 若首次触发该 `key`，进入轻伤，记录 `dangerMemory[key]=true`，扣轻伤数值并弹窗。
5. 若同一天再次触发同一 `key`，先按 `fatalChance` 判定是否致命；未致命才再次轻伤。

### 2.1.6 数值单位约定

- 文档中的 `3.5%` 在代码里写作 `0.035`。
- `+12 到 +22` 表示 `randomGain(12, 22, multiplier)`，上下限都可能取到。
- `* 某倍率` 表示该数值会受鼠鼠性格、主人信任或地点倍率影响。
- `最高 14%` 表示 `maxChance=0.14`，最终概率不会超过这个值。
- 低值类属性扣到 `0` 会触发死亡检查；出恭增加到 `100` 会触发死亡检查。

## 3. 时间流逝

- 游戏小时按 `0-23` 循环。
- 主人作息：`9:00-18:00` 上班，其余时间在家。
- 若主人 call 主人失败并被老板抓包，主人会一直不在家，直到 `0:00` 才回家。
- 每小时基础变化会受鼠鼠性格倍率影响：

| 属性 | 基础每小时变化 | 性格倍率 |
| --- | ---: | --- |
| 饱腹 | `-3` | `foodDecay` |
| 磨牙 | `-2` | `chewDecay` |
| 闲游 | `-3` | `playDecay` |
| 抚触 | `-3` | `loveDecay` |
| 出恭 | `+7` | `poopRate` |

- 主人在家且抚触值 `< 10` 时，会自动抚触，每次基础加 `20 * loveGain * 信任petGain`，消耗 `1` 次每日抚触次数，直到抚触不低于 `10` 或次数用完。
- 每天 `6:00` 进入新一天，刷新日记、邻居鼠故事、今日零食、危险记忆、每日次数、电话、主人相关日统计。
- 新一天每日次数重置为：吃饭 `11`、磨牙 `6`、闲游 `13`、电话 `2`；抚触上限按主人信任和出逃情况重新计算。

### 3.1 时间胶囊

- 点击主界面的”时钟”会打开二级时钟界面；界面中显示当前时间胶囊数量。
- 时间胶囊数量为 `0` 时，”跳过3小时”按钮禁用，只提示去笼子外偶尔能捡到。
- 时间胶囊数量 `> 0` 时，点击”跳过3小时”会消耗 `1` 颗，并让游戏时间连续推进 `3` 个小时，**五项生命值按正常每小时衰减计算**（饱腹-9、磨牙-6、闲游-9、抚触-9、出恭+21）。
- 胶囊跳时不会触发”突发事件”：不进行每小时的随机事件判定（低抚触自动摸摸、门外冻伤风险、门外捡食/出恭），也不会增加门外连续停留时间，避免玩家用加速反而被随机事件惩罚。
- 胶囊跳时会正常推进”时间相关规则”：主人上下班、老板抓包到 `0:00` 回家、禁足剩余小时递减、`6:00` 新一天刷新、每日次数刷新、邻居鼠故事刷新、今日零食刷新、补粮、抚触上限和信任恢复/中断都会正常发生。
- 时间胶囊为本局资源，重开后清零，不每日清零；持有上限 `5` 颗。

| 获得地点 | 掉落条件 | 基础概率 |
| --- | --- | ---: |
| 客厅 | 笼子外行动成功且未被主人抓包 | `3%` |
| 厨房 | 笼子外行动成功且未被主人抓包 | `4.5%` |
| 阳台 | 笼子外行动成功且未被主人抓包 | `4%` |
| 门外 | 笼子外行动成功且未被主人抓包 | `6%` |

- 掉落统一在行动结算入口处理；笼内吃饭、出恭、睡觉、整理窝、梳毛、跑轮不会掉落，地点移动本身也不会掉落，避免来回走路刷资源。
- 被主人发现并拎回笼子视为本次外出行动中断，不会获得时间胶囊。
- 触发掉落时底部日志写入“时间胶囊+1”，弹窗不展示额外数值。
- 音效：捡到时间胶囊播放 `timeCapsuleGet`（`mixkit-bubble-pop-alert-2357.mp3`）；使用时间胶囊播放 `timeCapsuleUse`（`mixkit-video-game-mystery-alert-234.mp3`）并补一声短促钟表滴答；无胶囊保护分支播放轻点击音 `timeCapsuleEmpty`。

## 4. 死亡线与预警

- 预警阈值：饱腹/磨牙/闲游/抚触 `<= 20` 时提醒；出恭 `>= 85` 时提醒。
- 死亡条件：
  - 饱腹 `<= 0`：饿死。
  - 磨牙 `<= 0`：牙齿问题死亡。
  - 闲游 `<= 0`：憋坏死亡。
  - 抚触 `<= 0`：第 `1` 天不会死，从存活天数 `>= 1` 开始死亡。
  - 出恭 `>= 100`：憋死。
  - 特殊死法：猫、拖鞋、水淹、冻死、噎住、电死、卡住/缠住、压住/夹住、摔落、中毒、中暑。
- 电死且已存活至少 `1` 天时，可以尝试一次心肺复鼠。
- 心肺复鼠成功率 `50%`；成功后回到笼子，饱腹/磨牙/闲游设为 `15`，出恭保持原值；这类室内兜底恢复按主人相关规则结算，主人在家时抚触最低恢复到 `15`，主人不在家时不增加。

## 6. 主人信任

- 初始信任：`80`。
- 信任影响每日补粮、每日抚触上限、主人抚触收益和被发现概率。
- 连续乖巧恢复信任：当日主人发现 `0`、溜进客厅 `<= 1`、出家门 `0`、轻伤 `0` 时算乖巧；第 `1` 个乖巧日信任 `+5`，连续 `>= 2` 个乖巧日信任 `+10`。

| 信任范围 | 标签 | 补粮惩罚 | 抚触上限惩罚 | 抚触收益倍率 | 被发现风险倍率 |
| --- | --- | ---: | ---: | ---: | ---: |
| `85-100` | 亲密 | `0` | `0` | `1.08` | `0.9` |
| `70-84` | 稳定 | `0` | `0` | `1` | `1` |
| `50-69` | 起疑 | `12` | `1` | `0.94` | `1.08` |
| `30-49` | 警惕 | `26` | `3` | `0.84` | `1.18` |
| `0-29` | 戒备 | `42` | `5` | `0.72` | `1.32` |

- 每日补粮公式：`max(75, baseStock - foodPenalty)`。
- 每日抚触上限公式：`max(5, baseLimit - lovePenalty)`。
- 正常新日补粮基准 `150`。
- 食物储备上限 `180`；厨房叼走小零食和电话补给都会受该上限限制。
- 当日溜进客厅次数 `> 2` 时，补粮基准随机从 `100-149` 中取值，再扣信任惩罚。
- 当日溜进客厅次数 `> 4` 时，次日抚触基准改为 `8`；否则为 `11`。

## 7. 主人发现与电话

- 主人发现只在主人在家时发生；主人不在家时静默跳过发现检查，不刷“未被发现”日志。
- 地点发现倍率：笼子 `0`、客厅 `1`、厨房 `2.6`、阳台 `1.35`、门外 `1.65`。
- 普通发现概率：`3.5% * 鼠鼠ownerRisk * 信任discoveryRisk * 地点discoveryRisk`，最高 `28%`。
- 普通发现惩罚：闲游 `-10 到 -16`、抚触 `-2 到 -6`；当天第 `1` 次发现信任 `-6`，第 `2` 次起信任 `-9`。
- 厨房不再使用“特殊发现 + 普通发现”双层体系，而是走同一套主人发现弹窗和惩罚，只使用更高风险参数：`4.5% * 1.28 * 鼠鼠ownerRisk * 信任discoveryRisk * 厨房discoveryRisk`，最高 `42%`。
- 厨房发现惩罚：闲游 `-12 到 -22`、抚触 `-4 到 -9`；当天第 `1` 次厨房发现信任 `-10`，第 `2` 次起信任 `-13`。
- 被主人发现后会记录“被主人发现”和“被主人拎回笼子”，当前位置立即改回 `笼子`，本次外出路线中断；弹窗文案按地点和具体行为生成，例如舔水珠、钻橱柜、翻厨余不会再共用固定腮帮子文案。
- 抓包后触发禁足机制：当天第 `1` 次抓包禁足 `2` 小时，重复抓包追加 `3` 小时，剩余禁足最高 `6` 小时。
- 禁足只限制“从笼子溜进客厅”这一条外出入口；笼内吃饭、出恭、睡觉、整理窝、梳毛、跑轮仍可使用，避免玩家被惩罚后完全无法恢复。
- 禁足期间地点按钮会显示“先乖一会儿”，地图上客厅会标记“禁足”；无论从按钮还是地图点击，都会被同一个移动入口拦截，不能绕过。
- 禁足每经过 `1` 个游戏小时减少 `1` 小时；到 `6:00` 新一天时直接清零，表示主人重新给鼠鼠一次机会。
- 抓包弹窗只显示事件和鼠鼠状态，不展示具体扣值；闲游、抚触、信任和禁足小时数只写入底部日志/主人记录。
- 若一次行动已经触发主人摸摸或门外好心人摸摸，本次行动会跳过主人发现检查，避免同一行动里同时出现“被宠”和“被抓”。
- 主人在家时不能 call 主人。
- 电话每日 `2` 次；同一天成功/失败拨打后 `ownerCalledHome=true`，不能再次 call。
- 电话失败：概率 `1/3`，主人被老板抓包，加班到 `0:00`，当天无法回家。
- 电话成功：主人提前回家；抚触增加 `10-50 * 信任petGain`；食物储备增加 `10-50 * max(0.65, 1 - foodPenalty / 140)`，食物储备上限 `180`。

## 14. 零食图鉴

- 零食池共 `54` 种：谨慎食用 `10`、果蔬 `15`、坚果 `4`、甜点零食 `17`、主食 `8`。
- 每局每天随机一个今日零食；当天只能选择“吃”或“不吃”一次。
- 不吃：当天零食机会用掉，不改变数值。
- 吃过后会点亮图鉴，记录风险：危险、谨慎或安全。
- 图鉴显示分类顺序：果蔬类、坚果类、甜点零食类、主食类；谨慎食用的外观归到果蔬类展示，吃过才知道危险。

## 15. 邻居鼠与自言自语

- 邻居鼠故事每 `4` 小时刷新一次，打开邻居鼠面板时也会检查是否需要刷新。
- 每个故事周期只有 `1` 次“吱吱”机会。
- 邻居鼠每日最多回信 `8` 次。
- 每次吱吱有 `35%` 概率收到回信；成功回信会让闲游 `+10`。
- 鼠鼠自言自语：开局第一句固定为“随意点点有惊喜哦”；之后 `25%` 概率说性格台词，约 `25.5%` 概率说引导台词，其余说普通台词。
- 每小时额外随机出现 `1-4` 次气泡，每次延迟 `0-30` 秒。
- 点击中央鼠鼠只算自己挠痒，增加累计挠挠次数，不增加抚触。

## 16. 死亡后鼠生总结

- 死亡时调用 `die(msg, deathType)`，会在设置 `gameOver=true` 后立刻生成 `lifeSummarySnapshot`。快照冻结当时的鼠鼠身份、死亡原因、存活天数、死亡时间、地点、最终五项属性、主人信任、食物记录、行动记录和邻居鼠互动记录。
- 死亡弹窗保留主按钮“我还要再活五年！”，点击后直接 `restart()` 开新局；旁边新增“鼠生总结”按钮，只打开本局备忘录，不自动重开。
- “鼠生总结”打开 `lifeSummaryOverlay`，界面按胶片走马灯分帧展示：片头、吃播记录、干过的事、邻居鼠通讯录、主人观察报告、鼠生结语。
- 胶片底部“先收起来”只关闭总结；“鼠生再来”先关闭总结，再调用 `restart()` 进入下一轮。
- 若电死触发“心肺复鼠”，死亡时仍会生成临时总结；如果救活成功，`continueGame()` 会清空 `lifeSummarySnapshot`，本局继续记录；如果直接重开，`restart()` 会清空上一局统计。

### 16.1 `lifeStats` 字段

| 字段 | 含义 | 写入时机 |
| --- | --- | --- |
| `actions` | 本局行动计数表，键为行动名，值为次数 | 按钮行为成功开始执行时记录；危险致死前已开始执行的动作也算一次 |
| `foods` | 本局吃过/吞过/尝过的食物汇总 | 吃饭、每日零食、偷零食、翻垃圾找到或吃坏东西、门外捡食等触发 |
| `foodTimeline` | 最近最多 `24` 条食物时间线 | 每次 `recordLifeFood()` 追加；超过上限丢弃最旧记录 |
| `neighborSqueaks` | 对邻居鼠“吱吱”的次数 | 每个故事周期首次点击邻居鼠按钮时记录 |
| `neighborReplies` | 收到邻居鼠回信次数 | 回信概率成功时记录 |
| `neighborMisses` | 邻居鼠未回应次数 | 回信次数用完或概率未命中时记录 |
| `neighborLastReply` | 最后一条邻居鼠回信文本 | 成功收到回信时覆盖 |
| `ownerDiscoveries` | 本局被主人发现总次数 | `checkOwnerDiscovered()` 统一记录；厨房也走同一函数 |
| `injuries` | 本局轻伤/逃过一劫总次数 | `applyDangerInjury()` 或轻微触电等中间态记录 |
| `escapes` | 本局溜进客厅总次数 | 从笼子移动到客厅时记录 |
| `outsideEscapes` | 本局出家门总次数 | 移动到门外时记录 |
| `ownerPets` | 主人摸摸次数 | 只有主人在家、自动抚触真实发生时记录 |
| `strangerPets` | 门外好心人摸摸次数 | 门外好心人摸摸事件触发时记录；陌生人的摸摸可直接增加抚触，不受主人在家状态限制 |

### 16.2 统计函数

| 函数 | 作用 |
| --- | --- |
| `createLifeStats()` | 创建新一局的空统计对象 |
| `ensureLifeStats()` | 防止旧存档或重开边界缺少统计对象 |
| `recordLifeAction(actionName, amount)` | 给行动计数；`amount` 默认为 `1` |
| `recordLifeFood(foodName, detail)` | 记录食物名称、次数和最多 `3` 条效果说明 |
| `recordLifeMetric(metricName, amount)` | 记录本局累计指标，如轻伤、出门、摸摸 |
| `recordNeighborLife(kind, text)` | 统一记录邻居鼠吱吱、回信和落空 |
| `createLifeSummarySnapshot(deathMsg, deathType)` | 死亡瞬间把可变状态整理成不可变快照 |
| `renderLifeSummary(snapshot)` | 根据快照生成胶片分帧 UI |

### 16.3 统计口径

- 行动记录不改变游戏数值，只服务于死亡总结。
- `spendPlayAction(actionName)` 会自动记录大多数消耗闲游次数的行为；笼内吃饭、出恭、睡觉、整理窝、梳毛，以及客厅旧行为“咬电线/翻垃圾/钻下水道/云游四方”单独补记。笼内“梳毛”不消耗闲游，使用 `dayGroomSelfCount` 限制每日有效收益。
- 食物记录只记录鼠鼠实际吃到、吞到、尝到或捡到的内容；“拒绝零食”只记为行动，不进入食物列表。
- 主人相关总结使用本局累计值，不读每日字段，避免存活多天后前几天记录丢失。
- 主人摸摸统一经过 `addLoveWhenOwnerHome()`，只有主人在家才会增加抚触；门外好心人摸摸经过 `addStrangerLove()`，和主人状态无关，触发即可直接增加抚触。
